Detailed Description

Core types and functions for the Dining Philosophers simulation.

Defines philosopher entities, table configuration, and all essential synchronization and lifecycle routines.

◆ t_philo

t_philo

#include <include/philo.h>

Represents a single philosopher in the simulation.

Contains:

  • id: Unique identifier of the philosopher (0-based index).
  • meal_count: Number of meals eaten.
  • left_fork: Index of the left fork.
  • right_fork: Index of the right fork.
  • last_meal: Timestamp of the last meal in milliseconds.
  • table: Pointer to the shared table structure.
  • thread: Thread handle running this philosopher's routine.

◆ t_table

t_table

#include <include/philo.h>

Configuration and global state shared by all philosophers.

Contains:

  • Simulation parameters like timeouts and counts.
  • Philosopher list and synchronization primitives.
  • Flags for termination, meal tracking, and start time.

◆ dinner_routine()

void * dinner_routine ( void *  arg)

#include <include/philo.h>

Entry point for each philosopher thread.

Simulates the life of a philosopher through an infinite loop of: thinking, picking up forks, eating, sleeping. Handles the case of even/odd IDs and philosopher count for timing offset.

Note
Terminates early if only one philosopher exists, or if the dinner has ended (is_dinner_over returns true).
Parameters
argA pointer to the philosopher's t_philo struct.
Returns
Always returns NULL.

◆ print_action()

void print_action ( t_philo *  philo,
const char *  action 
)

#include <include/philo.h>

Print a philosopher's current action with a timestamp.

Outputs a log line including:

  • The time since simulation start
  • The philosopher ID
  • The action string (e.g., "is eating", "has taken a fork")

Uses a mutex to ensure output remains synchronized across threads. Special-case: if action starts with ‘'e’`, prints END_MSG after the line.

Parameters
philoPointer to the philosopher who is performing the action.
actionString representing the action being performed.
